## Ask a question or report a bug

Use the repository's **Question**, **Bug report**, or **Feature request** template. Search existing
issues first, choose the closest template, and
include:

- what you were trying to do;
- the smallest sequence that reproduces the problem;
- your operating system and Python version;
- the EvalForge version or commit; and
- sanitized error text or screenshots.

Never attach API keys, access tokens, private prompts, benchmark content, model responses, database
files, or unredacted evidence exports.

Feature requests are easiest to evaluate when they explain the core evaluation problem, the people
affected, and how the idea keeps the test-set -> candidates -> results workflow simple.

## Security and conduct

Security vulnerabilities must follow [the security policy](SECURITY.md), not the public bug form.
Community concerns should follow the [Code of Conduct](CODE_OF_CONDUCT.md).
