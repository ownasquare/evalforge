# EvalForge v0.2.0

EvalForge v0.2.0 adds a permanently offline workflow for turning versioned JSON or CSV human-label
manifests into reviewable threshold-calibration evidence.

## Quick start

From a source checkout with dependencies installed:

```bash
uv run evalforge calibrate examples/calibration-labels.json \
  --threshold 0.7 \
  --output-dir ./private-calibration
```

The included JSON and CSV examples represent the same calibration set. Manifests are strict and
bounded, with explicit schema, dataset ID/version/SHA-256, metric name/version/direction, item ID,
score, human decision, and opaque reviewer ID fields. Invalid, unknown, duplicate, or inconsistent
input is rejected.

Reports are canonical and deterministic, written privately with restrictive permissions, and
published idempotently by payload hash. They include the selected threshold, confusion matrix,
precision, recall, F1, sample/pass/fail/reviewer counts, dataset identity, label-manifest SHA-256,
calibration-set SHA-256, and report-payload SHA-256.

## Evidence boundaries

The bundled fixtures demonstrate the file contract; they are not evidence of real human review.
This command does not collect labels, establish reviewer agreement, select or approve a threshold,
contact a model provider, certify hosted OIDC, or certify a production deployment. The report is
explicitly marked `evidence_kind: offline_statistical_evidence` and `production_validated: false`.

EvalForge v0.2.0 is not published to PyPI or an OCI registry. The GitHub Release is the distribution
and integrity boundary for this version.
