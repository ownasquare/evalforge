# EvalForge v0.2.0 release completion

Date: 2026-07-18 PDT (published 2026-07-19 UTC)

## Outcome

EvalForge v0.2.0 is published as a public, immutable prerelease. It adds the optional, private,
offline calibration workflow for reviewer-provided JSON or CSV labels while keeping the primary
dashboard workflow focused on test sets, candidate comparison, and result review.

The first annotated-tag push completed the draft-first release workflow without manual repair,
asset replacement, or republishing. This closes the release backlog item to exercise the immutable
publication path on a real follow-up version.

## Proof boundaries and remaining lanes

The bundled labels are deterministic fixtures, not records of real human review. Generated
calibration reports remain `offline_statistical_evidence` with `production_validated = false`.
Version 0.2.0 does not claim:

- representative human sampling, reviewer agreement, or threshold approval;
- a paid-provider calibration run;
- hosted OIDC acceptance;
- production deployment or recovery certification; or
- PyPI or OCI publication.

Those external acceptance lanes remain tracked in
[issue #2](https://github.com/ownasquare/evalforge/issues/2),
[issue #3](https://github.com/ownasquare/evalforge/issues/3), and
[issue #4](https://github.com/ownasquare/evalforge/issues/4). An optional persisted calibration
API or UI is also a future product decision; it is not required for the private CLI-first workflow
released here.
