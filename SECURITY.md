# Security policy

## Reporting a vulnerability

Please use the repository host's private vulnerability-reporting feature when available. Do not put
secrets, exploit payloads, proprietary evaluation data, or provider response bodies in a public
issue. Include the affected version, impact, minimal reproduction, and suggested mitigation.

## Supported version

The latest commit on the default branch is the supported development version. This portfolio project
does not yet publish long-term-support releases.

## Scope reminder

EvalForge is loopback-first and single-user. Network-exposed, multi-tenant, or compliance-regulated
deployments require authentication, authorization, TLS, tenant isolation, durable job execution, and
environment-specific security review beyond this repository's current claim.
